JARVIS: THE COMPLETE PICTURE
================================================================================

Jarvis is a free, intelligent desktop automation agent that starts as a hybrid 
context system (DOM + screenshots) and evolves into a predictive state machine 
through Pinecone-powered learning. It combines multi-key API fallback (4 Gemini 
keys), web/OS automation (Selenium + pyautogui), universal app launching, and 
silent vector memory to create a personal AI that gets smarter with every 
task—without screenshots slowing it down.


What Makes Jarvis Different
================================================================================

• Starts simple (screenshots), learns to predict (no screenshots)
• Uses free Gemini API with 4-key fallback strategy
• Silent learning (Pinecone vector memory, no notifications)
• Gets faster over time (fewer screenshots, better predictions)
• Runs on weak PCs (lightweight, hotkey-triggered)
• No coding needed (natural language automation)

1. API MANAGEMENT (4 GEMINI KEYS)
================================================================================

STEP 5: API Management (Updated)

• Storage: .env file (plaintext, local only)
• Keys: 4 Gemini API keys (GEMINI_KEY_1, GEMINI_KEY_2, GEMINI_KEY_3, GEMINI_KEY_4)
• Health checks: On startup AND whenever a key fails
• Fallback order: Gemini 1 → Gemini 2 → Gemini 3 → Gemini 4
• Timezone: EST for daily reset boundaries
• Daily quota per key: ~1M tokens (free tier)
• No token tracking: Log usage but don't calculate (overhead)

Fallback Logic
================================================================================

On API call failure (timeout, rate limit, error): Immediately move to next 
Gemini key. Log which key failed and at what time. Continue seamlessly—user 
doesn't notice. If all 4 keys fail, ask user for manual input. Reset tracking 
daily at midnight EST.

3. HYBRID → PREDICTIVE CONTEXT SYSTEM
================================================================================

STEP 25: Context Optimization (Hybrid to Predictive State Machine)

Jarvis starts with hybrid context (DOM extraction for web, screenshots for OS) 
and evolves into a predictive state machine through continuous learning. Over 
time, screenshots are replaced by accurate predictions based on historical 
action outcomes, dramatically improving speed and token efficiency.

Phase 1: Hybrid Data Collection (Tasks 1-20)
================================================================================

• Web tasks: Extract DOM (HTML structure) instead of always screenshotting
• OS tasks: Screenshot (necessary for pixel-level tasks)
• Collect transition data: Action → Before state → After state
• Store in Pinecone: Action type, before/after DOM/screenshot, success result
• Build baseline: Screenshots still used for analysis, but data is captured
• No user impact: Process is silent, user still gets results

Phase 2: Prediction Learning (Tasks 21-100)
================================================================================

• After 20+ similar actions recorded: Query Pinecone for pattern
• Analyze: What happens when user clicks button X? Scrolls down? Types text?
• Predict outcome: 'This click will show dropdown with 5 items'
• Confidence threshold: >85% similarity = predict without screenshot
• Fallback: If confidence <85%, take screenshot and verify
• Continuous improvement: Compare prediction to actual, adjust confidence
• Result: By task 100, ~40-50% of actions execute without screenshots

Phase 3: Full Predictive (Tasks 100+)
================================================================================

• Mature model: Thousands of action outcomes recorded
• High confidence: Most common actions predicted accurately (>90% confidence)
• Rare/new actions: Still use screenshots (necessary unknowns)
• Performance: ~70% fewer screenshots than initial phase
• Speed: Actions execute 2-3x faster
• Tokens: 60-70% fewer API calls (no screenshot analysis needed)

Data Collected per Action
================================================================================

For each executed action, Jarvis stores: action type (click/type/scroll/navigate), 
action target (element/coordinate), before state (DOM hash, screenshot hash), 
after state (DOM hash, screenshot hash), screenshot difference score, DOM changed 
(boolean), action success (boolean), task type (web/os), timestamp, and execution 
duration.

Prediction Algorithm
================================================================================

On new action: (1) Embed action description as vector, (2) Query Pinecone for 
top-5 similar past actions, (3) Filter for success=true outcomes, (4) If 3+ 
matches with >0.85 similarity, aggregate predicted results and return with 
confidence score, (5) If confidence ≥85%, use prediction; otherwise, take 
screenshot and verify.

Fallback & Verification
================================================================================

• Always take screenshot if: action type is rare, target is new, confidence <85%
• Compare prediction to actual: Calculate diff between predicted and actual state
• If diff <10%: Prediction was accurate, increase confidence for this pattern
• If diff >10%: Prediction was wrong, flag for more data collection, lower confidence

Real Impact on Performance
================================================================================

First Reddit scrape (Tasks 1-5): 4 actions, 5 screenshots, 45 seconds, ~50 tokens
Mid-phase (Tasks 30-50): 4 actions, 2 screenshots, 25 seconds, ~25 tokens
Mature (Tasks 150+): 4 actions, 0-1 screenshots, 12 seconds, ~10 tokens
Improvement: 75% faster, 80% fewer tokens, same accuracy

STEP 26: Confidence Scoring & Learning Loop
================================================================================

• Confidence = (successful_predictions / total_predictions) for similar actions
• Update in Pinecone: Store confidence_score in metadata for each action pattern
• Decay over time: If page structure changes, older data weighted less
• Feedback loop: Failed predictions trigger immediate screenshot verification
• Silent adjustment: Confidence scores update without user notification

STEP 27: Context Selection Strategy
================================================================================

• Web task (Selenium): Use DOM extraction (HTML), only screenshot if prediction fails
• OS task (pyautogui): Always screenshot (visual context needed)
• Mixed task: Use DOM for web part, screenshot for OS part
• Rare action: Always screenshot first time, predict future
• Safety: Always fallback to screenshot if anything uncertain
